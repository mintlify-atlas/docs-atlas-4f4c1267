# Features & Integrations

Schoolor Backend provides a rich set of features powered by modern integrations and cloud-native technologies. This guide covers the key capabilities across all three microservices.

## Authentication & Authorization

### SuperTokens Integration

Schoolor uses **SuperTokens** for modern, secure authentication:

```yaml
supertoken:
  uri: https://auth.example.com
  cdi-version: 2.20
  api-key: ${AUTH_API_KEY}
```

**Features:**
- Session management with refresh tokens
- JWT-based authentication
- Multi-tenant support
- Social login support
- Email/password authentication

### JWT Validation

All services validate JWTs using SmallRye JWT:

**Development:**
```yaml
mp:
  jwt:
    verify:
      publickey:
        location: https://supertokens.example.com/.well-known/jwks.json
```

**Production:**
```yaml
mp:
  jwt:
    verify:
      publickey:
        location: /keys/publickey-prod.pem
```

<Note>
  In development, JWT keys are fetched from SuperTokens JWKS endpoint. In production, use local PEM files for better performance and reliability.
</Note>

### Example: Protected GraphQL Query

```kotlin
@GraphQLApi
class UserResource {
    
    @Query
    @RolesAllowed("user", "admin")
    fun getCurrentUser(@Context jwt: JsonWebToken): User {
        val userId = jwt.subject
        return userService.findById(userId)
    }
}
```

### Webhook Handling

Stripe webhooks are validated using the endpoint secret:

```kotlin
@POST
@Path("/webhooks/stripe")
fun handleStripeWebhook(
    @HeaderParam("Stripe-Signature") signature: String,
    payload: String
): Response {
    val event = Webhook.constructEvent(
        payload, 
        signature, 
        stripeEndpointSecret
    )
    
    when (event.type) {
        "payment_intent.succeeded" -> handlePaymentSuccess(event)
        "customer.subscription.updated" -> handleSubscriptionUpdate(event)
        // ...
    }
    
    return Response.ok().build()
}
```

<Warning>
  Always validate webhook signatures to ensure events are genuinely from Stripe and not spoofed.
</Warning>

## Messaging & Notifications

### MQTT Messaging

All services use **MQTT** for asynchronous, event-driven communication:

```yaml
mqtt:
  host: ${MQTT_HOST}
  port: ${MQTT_PORT:1883}
  connector: smallrye-mqtt
  enabled: ${MQTT_ENABLED:true}
```

**Use Cases:**
- Real-time chat message delivery
- File upload notifications
- Cross-service event propagation
- Lesson request notifications

### SmallRye Reactive Messaging

MQTT integration uses SmallRye Reactive Messaging:

```kotlin
@ApplicationScoped
class ChatMessageProcessor {
    
    @Incoming("chat-messages")
    @Outgoing("chat-messages-out")
    fun processMessage(message: ChatMessage): ChatMessage {
        // Process and transform message
        logger.info { "Processing message: ${message.id}" }
        return message.copy(processed = true)
    }
}
```

### Shared Subscriptions

For load balancing across multiple instances:

```yaml
topic: $share/g/chat-message/#
```

The `$share/g/` prefix enables MQTT shared subscriptions, distributing messages across consumer instances.

<Info>
  Shared subscriptions ensure only one instance processes each message, enabling horizontal scaling without duplicate processing.
</Info>

### File Upload Flow

```kotlin
@POST
@Path("/upload")
@Consumes(MediaType.MULTIPART_FORM_DATA)
fun uploadFile(
    @MultipartForm fileUpload: FileUpload,
    @Context jwt: JsonWebToken
): FileMetadata {
    // Validate file size and type
    validateFile(fileUpload)
    
    // Generate unique filename
    val fileName = "${UUID.randomUUID()}-${fileUpload.fileName}"
    
    // Upload to MinIO
    val url = minioService.upload(
        bucket = "schoolr",
        objectName = fileName,
        inputStream = fileUpload.data,
        contentType = fileUpload.contentType
    )
    
    // Publish MQTT event
    fileEventPublisher.publishUpload(fileName, url, jwt.subject)
    
    return FileMetadata(
        id = UUID.randomUUID().toString(),
        fileName = fileName,
        url = url,
        size = fileUpload.size,
        uploadedBy = jwt.subject
    )
}
```

## Real-time Chat

### GraphQL Subscriptions

Real-time message updates using GraphQL subscriptions:

```kotlin
@GraphQLApi
class ChatSubscriptions {
    
    @Subscription
    fun onNewMessage(@Name("roomId") roomId: String): Multi<Message> {
        return messageService.subscribeToRoom(roomId)
    }
}
```

**Client Example:**

```graphql
subscription OnNewMessage($roomId: ID!) {
  onNewMessage(roomId: $roomId) {
    id
    content
    senderId
    timestamp
    metadata {
      attachments
      edited
    }
  }
}
```

### Message Persistence

All chat messages are persisted in PostgreSQL:

```kotlin
@Entity
@Table(name = "messages")
class Message : PanacheEntity() {
    
    @Column(nullable = false)
    lateinit var roomId: String
    
    @Column(nullable = false)
    lateinit var senderId: String
    
    @Column(columnDefinition = "TEXT")
    var content: String? = null
    
    @Column(nullable = false)
    lateinit var timestamp: Instant
    
    @OneToOne(cascade = [CascadeType.ALL])
    var metadata: MessageMetadata? = null
}
```

### Room Management

**Features:**
- Create private and group rooms
- Add/remove subscribers
- Room metadata (name, avatar, settings)
- Message history pagination
- Unread message counts

```kotlin
fun createRoom(name: String?, subscriberIds: List<String>): Room {
    val room = Room().apply {
        this.name = name
        this.createdAt = Instant.now()
    }
    room.persist()
    
    // Add subscribers
    subscriberIds.forEach { userId ->
        Subscriber().apply {
            this.roomId = room.id.toString()
            this.userId = userId
            this.joinedAt = Instant.now()
        }.persist()
    }
    
    return room
}
```

## Database Management

### Liquibase Migrations

Database schema is managed via **Liquibase** changelogs:

```yaml
quarkus:
  liquibase:
    migrate-at-start: true  # Dev only
    change-log: db/changelog/master.xml
```

**Directory Structure:**

```
db/
└── changelog/
    ├── master.xml
    ├── v1.0/
    │   ├── create-users-table.xml
    │   ├── create-lessons-table.xml
    │   └── ...
    └── v2.0/
        └── add-room-metadata.xml
```

### Multiple Databases

Two separate PostgreSQL databases:

**Data Management:**
```yaml
jdbc:
  url: jdbc:postgresql://localhost:5432/schoolr
```

**Chat:**
```yaml
jdbc:
  url: jdbc:postgresql://localhost:5432/schoolr-chat
```

<Info>
  Separate databases provide data isolation, independent scaling, and clearer service boundaries.
</Info>

## Scheduled Jobs

The Data Management service includes a job scheduler:

```yaml
scheduler:
  enabled: ${SCHEDULER_ENABLED:true}
```

**Use Cases:**
- Send lesson reminders
- Process pending payments
- Clean up expired sessions
- Generate daily reports
- Sync data with external services

**Example:**

```kotlin
@ApplicationScoped
class LessonReminderScheduler {
    
    @Scheduled(cron = "0 0 9 * * ?")  // Daily at 9 AM
    fun sendDailyReminders() {
        logger.info { "Sending daily lesson reminders" }
        val lessons = lessonService.getTodayLessons()
        lessons.forEach { lesson ->
            notificationService.sendReminder(lesson)
        }
    }
}
```

<Warning>
  In production, ensure only one instance has the scheduler enabled to avoid duplicate job execution, or use distributed locking.
</Warning>

## Configuration Profiles

### Development (`%dev`)

- CORS enabled for localhost
- Liquibase auto-migration
- GraphQL UI enabled
- Scheduler disabled (to avoid background jobs during development)
- SuperTokens dev endpoint
- Detailed logging

### Testing (`%test`)

- Dev Services enabled (auto-start databases)
- MQTT disabled
- Scheduler disabled
- In-memory or test databases

### Production (`%prod`)

- Environment-based configuration
- HTTP compression enabled
- CORS from `CORS_ORIGINS`
- JWT from local PEM file
- Liquibase manual migration (set `migrate-at-start: false`)
- JSON logging for log aggregation

<Info>
  Use environment variables extensively in production to keep secrets out of configuration files and enable environment-specific settings.
</Info>
