## Email Verification

### verifyEmailCode

Verify an email address using a one-time code sent to the user.

#### Request

```graphql
mutation VerifyEmailCode($code: ConsumeCodeDto!) {
  verifyEmailCode(code: $code) {
    email
    phone
    confirmedEmail
    confirmedPhone
    enabled
    tutorStatus
  }
}
```

#### Parameters

<ParamField body="code" type="ConsumeCodeDto" required>
  OTP verification code details
  
  <Expandable title="ConsumeCodeDto fields">
    <ParamField body="preAuthSessionId" type="String" required>
      Session ID from the sendEmailCode response
    </ParamField>
    <ParamField body="deviceId" type="String" required>
      Device ID from the sendEmailCode response
    </ParamField>
    <ParamField body="userInputCode" type="String" required>
      The OTP code entered by the user
    </ParamField>
  </Expandable>
</ParamField>

#### Response

Returns a `CheckAccountDto` with updated verification status.

#### Authentication

<Check>Requires authentication</Check>

#### Example

```json
{
  "code": {
    "preAuthSessionId": "abc123def456",
    "deviceId": "device789",
    "userInputCode": "123456"
  }
}
```

### sendEmailCode

Send a verification code to the user's pending email address.

#### Request

```graphql
mutation {
  sendEmailCode {
    status
    preAuthSessionId
    deviceId
    timeCreated
    codeLifetime
  }
}
```

#### Response

<ResponseField name="status" type="String">
  Status of the code sending operation
</ResponseField>

<ResponseField name="preAuthSessionId" type="String">
  Session ID to use when verifying the code
</ResponseField>

<ResponseField name="deviceId" type="String">
  Device ID to use when verifying the code
</ResponseField>

<ResponseField name="timeCreated" type="Long">
  Timestamp when the code was created (milliseconds)
</ResponseField>

<ResponseField name="codeLifetime" type="Long">
  How long the code is valid (milliseconds)
</ResponseField>

#### Authentication

<Check>Requires authentication</Check>

#### Notes

- Sends code to the `email_to_be_confirmed` field
- Code is typically valid for 5-15 minutes

## Parent Account Management

All parent-related mutations require the authenticated user to be a student account with proper authorization to manage the specified parent account. The `parentUUID` parameter identifies which parent account to operate on.

### Parent Account Flow

1. Student creates or links a parent account
2. Parent email/phone is set using `updateParentEmail` or `updateParentUserPhone`
3. Verification codes are sent using `sendParentEmailCode` or `sendParentPhoneCode`
4. Codes are verified using `verifyParentEmailCode` or `verifyParentPhoneCode`
5. Student can update parent profile with `updateParentAccount`
6. Student can reset parent password with `resetParentPassword`

### Security Notes

- Only the linked student can manage their parent accounts
- Parent accounts cannot manage themselves through these endpoints
- Each operation validates the parent-student relationship
