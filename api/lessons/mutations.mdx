## createLesson

Creates a new lesson in the system.

**Source**: `LessonResource.kt:124`

**Authentication**: Required (Admin access)

### Parameters

<ParamField path="data" type="CreateUpdateLessonDto" required>
  Lesson creation data:
  - `lessonDate` (LocalDate) - Date of the lesson
  - `lessonTime` (LocalTime) - Start time of the lesson
  - `status` (LessonStatus) - Initial status (typically PENDING)
  - `minutes` (BigInteger) - Duration in minutes
  - `subjectId` (BigInteger) - ID of the subject being taught
  - `classroomId` (BigInteger) - ID of the classroom
</ParamField>

### Returns

`Lesson` - The created lesson object

### Example

```graphql
mutation CreateNewLesson {
  createLesson(
    data: {
      lessonDate: "2026-03-15"
      lessonTime: "14:00:00"
      status: PENDING
      minutes: 60
      subjectId: 123
      classroomId: 456
    }
  ) {
    id
    uuid
    lessonDate
    lessonTime
    status
    minutes
    subject {
      id
      name
    }
    classroom {
      id
      signature
    }
  }
}
```

## bookLesson

Books a paid lesson for a student.

**Source**: `BookLessonResource.kt:36`

**Authentication**: Required

### Parameters

<ParamField path="data" type="BookLessonDto" required>
  Booking data:
  - `subjectId` (BigInteger) - ID of the subject
  - `lessonResponseUUID` (UUID) - UUID from lesson request
  - `classRoomSig` (String, required) - Classroom signature
  - `minutes` (Long) - Lesson duration (default: 0)
  - `lessonDate` (String) - Date in ISO format
  - `lessonTime` (String) - Time in ISO format
  - `message` (String) - Optional message for the tutor
  - `paymentMethod` (PaymentMode) - Payment method (default: SINGLE)
  - `friendIds` (List of Long) - IDs of friends joining
  - `homeLesson` (Boolean) - Whether it's a home lesson (default: false)
</ParamField>

### Returns

`BookLessonOutDto` - Booking result with payment and lesson details

### Example

```graphql
mutation BookPaidLesson {
  bookLesson(
    data: {
      subjectId: 123
      lessonResponseUUID: "123e4567-e89b-12d3-a456-426614174000"
      classRoomSig: "classroom-sig-123"
      minutes: 60
      lessonDate: "2026-03-15"
      lessonTime: "14:00:00"
      message: "Looking forward to the lesson!"
      paymentMethod: PAID
      friendIds: []
      homeLesson: false
    }
  ) {
    paymentTransactionId
    paymentTransactionUUID
    lessonId
    lessonUUID
    lessonResponseUUID
    paymentId
    paymentUUID
    total
    transactionAmount
    countTransactions
    enableVouchers
    isHomeLesson
  }
}
```

## Lesson State Transitions

The following diagram shows the valid lesson state transitions:

```
PENDING
  ├─→ CONFIRMED (via acceptLesson)
  ├─→ REJECTED (via refuseLesson)
  └─→ CANCELED (via cancelLesson)

CONFIRMED
  ├─→ COMPLETED (automatically after lesson ends)
  ├─→ CANCELED (via cancelLesson)
  └─→ RESCHEDULED (via moveLesson)

CANCELED, REJECTED, COMPLETED
  └─→ [Terminal states - no further transitions]
```

## Payment Mode Usage

<Card title="FREE" icon="gift">
  Used for first lessons, promotional lessons, or when a tutor offers a free session. Call `bookFreeLesson` mutation.
</Card>

<Card title="PAID" icon="credit-card">
  Standard paid lessons that require payment processing. Call `bookLesson` mutation with payment details.
</Card>

<Card title="SINGLE" icon="receipt">
  One-time payment for a single lesson (default mode). Used in `bookLesson` mutation.
</Card>
