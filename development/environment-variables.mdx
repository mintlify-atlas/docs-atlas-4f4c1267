Schoolr Backend services are configured via environment variables and YAML configuration files. This page documents all available environment variables across the three services.

## Overview

Environment variables follow this pattern:
- **Development**: Most variables have sensible defaults in `application.yaml`
- **Production**: All sensitive variables must be set via environment
- **Format**: `${VARIABLE_NAME:default_value}`

<Warning>
**Never commit sensitive values** like API keys, passwords, or secrets to the repository. Use environment variables or a `.env` file (which is in `.gitignore`).
</Warning>

## Common Variables

These variables are used across multiple services.

### Quarkus Configuration

<ParamField path="QUARKUS_INIT_AND_EXIT" type="boolean" default="false">
  If `true`, Quarkus initializes then exits immediately (useful for schema generation or validation).
</ParamField>

<ParamField path="LOG_LEVEL" type="string" default="INFO">

</ParamField>

<ParamField path="CORS_ORIGINS" type="string" required>
  **Production only**. Comma-separated list of allowed CORS origins.

</ParamField>

## Using Environment Variables

### .env File (Development)

Create a `.env` file in the project root (it's in `.gitignore`):

```bash
# Database
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_USER=schoolr-user
POSTGRES_PASSWORD=your-password

# Auth
AUTH_CONNECTION_URI=https://your-supertokens.io
AUTH_API_KEY=your-api-key

# Stripe
STRIPE_SECRET_KEY=sk_test_...
STRIPE_ENDPOINT_SECRET=whsec_...

# MinIO (for filemanager)
MINIO_HOST=http://localhost
MINIO_PORT=9000
MINIO_USERNAME=minioadmin
MINIO_PASSWORD=minioadmin
MINIO_BUCKET=schoolr

# MQTT
MQTT_HOST=localhost
MQTT_PORT=1883
MQTT_ENABLED=true
```

### Loading .env in Quarkus

Quarkus doesn't load `.env` by default. To use it:

**Option 1**: Export variables before running:

```bash
set -a && source .env && set +a
./mvnw quarkus:dev -pl apps/schoolr-datamanagement
```

**Option 2**: Use a startup script:

```bash
#!/bin/bash
# run-dev.sh
export $(cat .env | xargs)
./mvnw quarkus:dev -pl apps/schoolr-datamanagement
```

**Option 3**: IDE configuration (IntelliJ IDEA):
- Run → Edit Configurations
- Add "Environment variables" from `.env` file
- Or use the EnvFile plugin

### Docker / Kubernetes

For containerized deployments:

**Docker Compose**:

```yaml
services:
  datamanagement:
    image: schoolr-datamanagement:latest
    environment:
      - POSTGRES_HOST=postgres
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
      - STRIPE_SECRET_KEY=${STRIPE_SECRET_KEY}
    env_file:
      - .env.prod
```

**Kubernetes**:

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: schoolr-config
data:
  POSTGRES_HOST: "postgres-service"
  MQTT_HOST: "mqtt-broker"
---
apiVersion: v1
kind: Secret
metadata:
  name: schoolr-secrets
type: Opaque
stringData:
  POSTGRES_PASSWORD: "secure-password"
  STRIPE_SECRET_KEY: "sk_live_..."
```

## Troubleshooting

<Accordion title="Service starts but can't connect to database">
Check your database environment variables:

```bash
echo $POSTGRES_HOST
echo $POSTGRES_PORT
echo $POSTGRES_DB
echo $POSTGRES_USER
# Don't echo password in production!
```

Verify PostgreSQL is running:
```bash
psql -h $POSTGRES_HOST -p $POSTGRES_PORT -U $POSTGRES_USER -d $POSTGRES_DB
```
</Accordion>

<Accordion title="CORS errors in browser">
Check `CORS_ORIGINS` in production:

```bash
echo $CORS_ORIGINS
```

Ensure it includes your frontend URL (without trailing slash):
```bash
export CORS_ORIGINS=https://app.schoolr.net,https://schoolr.net
```
</Accordion>

<Accordion title="JWT verification fails">
**Development**: Ensure SuperTokens dev instance is accessible

**Production**: Verify `JWT_KEY_PATH` points to the correct PEM file:

```bash
ls -la $JWT_KEY_PATH
cat $JWT_KEY_PATH  # Should show PEM-formatted public key
```
</Accordion>

<Accordion title="File uploads fail (filemanager)">
Check MinIO configuration:

```bash
echo $MINIO_HOST
echo $MINIO_PORT
echo $MINIO_BUCKET

# Test MinIO connection
mc alias set myminio $MINIO_HOST $MINIO_USERNAME $MINIO_PASSWORD
mc ls myminio/$MINIO_BUCKET
```

Ensure the bucket exists:
```bash
mc mb myminio/$MINIO_BUCKET
```
</Accordion>

<Accordion title="Environment variables not loading">
Verify variables are exported:

```bash
env | grep POSTGRES
env | grep STRIPE
env | grep MINIO
```

If using `.env`, ensure you source it:
```bash
set -a && source .env && set +a
```
</Accordion>
