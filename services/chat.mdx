## Overview

The Chat Service is a dedicated microservice that handles real-time messaging functionality for the Schoolr platform. It manages chat rooms, messages, and real-time subscriptions using GraphQL and MQTT for message delivery.

## Service Configuration

### Port and Base URL

- **Port**: `9090`
- **Base URL**: `http://localhost:9090` (dev)
- **GraphQL Endpoint**: `/graphql`
- **GraphQL UI**: Enabled (configurable via `ENABLE_GRAPHQL_UI`)

Configuration reference (`application.yaml:3-4`):

```yaml
quarkus:
  http:
    port: 9090
```

### Database Configuration

The service uses a dedicated PostgreSQL database:

```yaml
datasource:
  db-kind: postgresql
  username: ${POSTGRES_USER:schoolr-user}
  password: ${POSTGRES_PASSWORD:password}
  jdbc:
    url: jdbc:postgresql://${POSTGRES_HOST:localhost}:5432/${POSTGRES_DB:schoolr-chat}
```

<Note>
The chat service uses its own database (`schoolr-chat`) separate from the main data management service for better scalability and isolation.
</Note>

### Message Management

The service provides comprehensive message handling:

- **Send Messages** - Create and publish messages to rooms
- **Message History** - Query historical messages with pagination
- **File Messages** - Handle messages with file attachments
- **Message Metadata** - Support for custom message metadata (images, files, etc.)
- **Real-time Delivery** - MQTT-based message broadcasting

### Real-time Subscriptions

GraphQL subscriptions enable real-time message delivery to clients:

```graphql
subscription {
  messageCreated(topic: "room-uuid") {
    uuid
    message
    senderUUID
    createdAt
    metadata {
      key
      value
    }
  }
}
```

## GraphQL API

The Chat Service exposes a single GraphQL resource: `ChatResource` (`src/main/java/it/simultech/chat/resources/ChatResource.kt`).

### Queries

| Query | Description | Parameters |
|-------|-------------|------------|
| `getRooms` | Get all available rooms | None |
| `getRoomByUUID` | Get a specific room by UUID | `uuid: String` |
| `getMessages` | Get messages from a room (paginated) | `room_uuid: String`, `index: Int`, `items: Int` |
| `getMessagesWithFiles` | Get messages containing files | `room_uuid: String`, `index: Int`, `items: Int` |
| `countMessagesWithFiles` | Count messages with file attachments | `room_uuid: String` |

### Mutations

| Mutation | Description | Input |
|----------|-------------|-------|
| `createMessage` | Send a new message to a room | `MessageDto` (room_uuid, message, metadata) |
| `createRoom` | Create a new chat room | `topic: String` |

Reference: `ChatResource.kt:72-82`

```kotlin
@Mutation
fun createMessage(messageInput: MessageDto): ChatMessageOutDto {
    val uuid = identityUtil.accountUUID ?: throw Exception("User UUID not provided")
    val message = messageService.publishMessage(messageInput = messageInput, uuid = uuid)
    return message.toMessageOut()
}
```

### Subscriptions

| Subscription | Description | Parameters |
|--------------|-------------|------------|
| `messageCreated` | Subscribe to new messages in a room | `topic: String` (room UUID) |

Reference: `ChatResource.kt:85-91`

```kotlin
@Subscription
fun messageCreated(topic: String): Multi<ChatMessageOutDto> {
    return processorsSingleton
        .getProcessor("chat-message/$topic")
        .toHotStream()
}
```

## MQTT Messaging

The service uses MQTT for scalable message distribution:

```yaml
mqtt:
  host: ${MQTT_HOST:34.154.145.9}
  port: ${MQTT_PORT:1883}
  connector: smallrye-mqtt
  enabled: ${MQTT_ENABLED:true}
```

### Message Channels

**Incoming Channels:**

| Channel | Topic | Purpose |
|---------|-------|---------|
| `chat-messages` | `$share/g/chat-message/#` | Messages from main service |
| `filemanager-messages` | `$share/g/chat-message-filemanager/#` | File upload notifications |
| `lessonrequest-messages` | `$share/g/chat-message-lessonrequest/#` | Lesson request updates |

**Outgoing Channels:**

| Channel | Purpose |
|---------|---------|
| `chat-messages-out` | Publish messages to MQTT broker |
| `chat-topic-messages-out` | Topic-specific message publishing |

<Note>
The `$share/g/` prefix enables MQTT shared subscriptions for load balancing across multiple service instances.
</Note>

## Message Flow

1. **Client sends message** via GraphQL mutation `createMessage`
2. **Message is persisted** to PostgreSQL database
3. **Message is published** to MQTT topic `chat-message/{room-uuid}`
4. **MQTT broker distributes** to all subscribed service instances
5. **Subscribers receive** via GraphQL subscription `messageCreated`

## Integration with Other Services

### File Manager Service

The chat service receives file upload notifications from the File Manager service:

- Topic: `chat-message-filemanager/#`
- Subscription: `filemanager-messages`
- Handler: `FileManagerSubscription.kt`

When a file is uploaded to a room, the File Manager publishes a message containing:
- `room_id` - Target chat room
- `message` - Description (e.g., "Nuovo file")
- `senderUUID` - User who uploaded the file
- `messageMetadata` - File path and metadata

Reference: File Manager `publishOnClassroom()` method

### Data Management Service

Receives lesson request notifications:

- Topic: `chat-message-lessonrequest/#`
- Subscription: `lessonrequest-messages`

## Authentication

The service uses JWT-based authentication with SuperTokens:

**Development:**
```yaml
mp:
  jwt:
    verify:
      publickey:
        location: https://dev-{id}.supertokens.io/.well-known/jwks.json
```

**Production:**
```yaml
mp:
  jwt:
    verify:
      publickey:
        location: ${JWT_KEY_PATH:keys/publickey-prod.pem}
```

The `IdentityUtil` service extracts the authenticated user's UUID from the JWT token for authorization.
