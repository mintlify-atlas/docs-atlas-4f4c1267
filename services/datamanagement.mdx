## Overview

The Data Management Service is the main backend API service for the Schoolr platform. It handles core business logic including user management, lesson scheduling, payments, tutoring features, and integrations with external services.

## Service Configuration

### Port and Base URL

- **Port**: `8080` (default)
- **Base URL**: `http://localhost:8080` (dev) / `https://api.schoolr.net` (prod)
- **GraphQL Endpoint**: `/graphql`
- **GraphQL UI**: `/graphql-ui` (always enabled)

### Database Configuration

The service uses PostgreSQL as its primary database:

```yaml
datasource:
  db-kind: postgresql
  username: ${POSTGRES_USER:schoolr-user}
  password: ${POSTGRES_PASSWORD:password}
  jdbc:
    url: jdbc:postgresql://${POSTGRES_HOST:localhost}:5432/${POSTGRES_DB:schoolr}
```

<Note>
The service uses Liquibase for database migrations. Migration files are located in `db/changelog/master.xml`.
</Note>

- **Lesson Responses** - Manage tutor responses to lesson requests

## MQTT Messaging

The service uses MQTT for real-time chat message notifications:

```yaml
mqtt:
  host: ${MQTT_HOST}
  port: ${MQTT_PORT:1883}
  connector: smallrye-mqtt
  enabled: ${MQTT_ENABLED:true}
```

**Topics:**
- `chat-message-notification/#` - Incoming chat messages
- `chat-messages-out` - Outgoing chat messages

## Environment Variables

Key environment variables for configuration:

| Variable | Purpose | Default |
|----------|---------|---------|
| `POSTGRES_HOST` | Database host | `localhost` |
| `POSTGRES_DB` | Database name | `schoolr` |
| `POSTGRES_USER` | Database user | `schoolr-user` |
| `CORS_ORIGINS` | Allowed CORS origins | Environment-specific |
| `JWT_KEY_PATH` | JWT public key path | `keys/publickey-prod.pem` |
| `STRIPE_SECRET_KEY` | Stripe API key | - |
| `NOVU_API_KEY` | Novu API key | - |
| `AUTH_CONNECTION_URI` | SuperTokens URI | - |
| `MQTT_HOST` | MQTT broker host | `34.154.145.9` |

<Warning>
The scheduler is disabled in development mode by default. Enable it with `SCHEDULER_ENABLED=true` for testing scheduled tasks.
</Warning>
